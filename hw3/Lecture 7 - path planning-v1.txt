What is Path Planning?

*Dynamics: the relationship between the
forces acting on a robot and the resulting
motion of the robot

• Path planning is the process of computing a feasible path from
an initial state to a target state, while satisfying constraints such
as:
• avoiding obstacles
• minimizing cost (distance or time)

• A pure geometric problem of finding a collision-free path without
concerning the dynamics*, the duration of motion, or constraints
on the motion*

Motion Planning concerns dynamics, duration of motion,
and constraints on the motion and the control inputs

Types of Path Planning
1. Graph-Based Methods
• Discretize space into nodes and edges
• Algorithms:
• Dijkstra’s Algorithm
• A* search algorithm

• Good for structured environments

2. Sampling-Based Methods
• Randomly explore space instead of full discretization
• Algorithms:
• Rapidly-exploring Random Tree
• Probabilistic Roadmap

• Scales well to high-dimensional robotics problems

Types of Path Planning
3. Optimization-Based Methods

• Formulate as minimizing a cost function
• Produces smooth, realistic trajectories
• Representative algorithms:
• Trajectory optimization
• Model predictive control (MPC)
• Potential field

4. Learning-Based Methods

• Use deep learning or reinforcement learning
• Learn policies from data or simulation
• Algorithms:
• Imitation learning
• Reinforcement learning
• Generative models

Path Planning as Search
• Task: find a path from start vertex to goal vertex given a graph
• Search Problem:
• State (configurations of the world)
• Initial state and goal test

• Action (go to the adjacent node, e.g., NSEW in a grid-like map)
• Step and path costs (e.g., 1)
• Transition model, also called Successor (i.e., St+1 =T(St, At))

• A solution is a sequence of actions (a plan) which transforms
the initial state to a goal state

• Heuristic functions are used to map every node in the graph to
a non-negative value, that represents the “best guess distance”
from that node to the goal
• for example, Euclidean distance (L2 distance)

Sampling based Planning
• Key idea:
• Rather than exhaustively explore ALL possibilities, randomly explore a
smaller subset of possibilities while keeping track of progress

• Search for a collision-free path only by sampling points

Probability Roadmap (PRM)
• Probabilistic Roadmap methods proceed in two phases :
• Preprocessing Phase – construct a roadmap G
• Query (Search) Phase –search a path from q_init and q_goal

• The roadmap is an undirected graph G = (N, E)
• The nodes in N are a subset of configurations of the robot sampled that
are collision free
• The edges in E correspond to feasible straight-line paths

General Idea of RRT Algorithm
• Search for a path from the initial configuration to the goal
configuration by expanding a search tree iteratively
• Instead of a general graph

• For each step
• samples a target configuration and expands the tree towards it
• The sample can either be a random configuration or the goal
configuration itself
